## 📝 Key Learnings

### Opposite Direction Two Pointers

- Left pointer starts from the beginning.
- Right pointer starts from the end.
- Used for sorted arrays and palindrome problems.
- If current sum is smaller than target → Move Left.
- If current sum is greater than target → Move Right.
- Time Complexity: **O(n)**
- Space Complexity: **O(1)**

---

### Same Direction (Fast & Slow)

- Fast pointer explores every element.
- Slow pointer tracks the next valid position.
- Used for in-place array modification.
- Fast explores.
- Slow builds.
- Time Complexity: **O(n)**
- Space Complexity: **O(1)**

---

### Valid Palindrome Concepts

- Ignoring spaces and special characters using `isalnum()`.
- Converting characters using `lower()`.
- Using `not` with Boolean expressions.
- Comparing characters using Two Pointers.

---

### Move Zeroes Concept

- Collect non-zero elements at the beginning.
- Swap non-zero elements with the slow pointer.
- Preserve relative order of non-zero elements.
- In-place modification without extra array.

## 🧠 Debugging Lessons Learned

- A single character (like starting a loop from `1` instead of `0`) can completely change an algorithm.
- Always dry-run custom test cases before submitting.
- Small syntax mistakes (left vs right pointer, `<` vs `<=`) can cause logical errors.
- Read the entire code once before submitting.
