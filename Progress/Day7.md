## 🧠 Learning & Reflection

Today was an important learning day rather than a problem-solving day.

### Key Realizations

- I understand the concepts but struggle to **reconstruct the solution** after a few days.
- My weakness is **algorithm recall**, not concept understanding.
- I tend to remember code instead of remembering the **algorithm story** behind each problem.
- Prefix Sum and other medium-level problems require stronger conceptual recall.
- I often default to using `for` loops without first analyzing the traversal pattern.

### New Learning Strategy

Instead of memorizing code, I will remember:

- Problem Trigger
- Pattern Used
- Algorithm Story
- Then reconstruct the code

Examples:

- Pair Sum → HashMap → Find Complement
- Duplicate → HashSet → Seen Before
- Sorted Search → Binary Search
- Range Sum → Take Everything → Remove Front
- Maximum Sum → Continue or Restart (Kadane)
- Maximum Product → Track Max & Min
- Merge Sorted Arrays → Fill From Back

## 💡 Important Concepts Learned

### Choosing Between `for` and `while`

**Use `for` when:**

- Traversing every element
- Python controls the next iteration
- Sequential traversal

**Use `while` when:**

- Manually moving pointers
- Binary Search
- Two Pointer problems
- Merge problems
- Unknown stopping condition

Golden Rule:

> If Python controls the traversal → `for`
>
> If I control the traversal → `while`
